# 🎯 ClientForFrameWork 前后端分离整改计划

## 一、核心目标

| 目标 | 说明 |
|------|------|
| **前后端分离** | UI全QML，业务逻辑全C++ |
| **TitleBar动态化** | 单组件=单层，多组件=双层（Chrome风格标签） |
| **Crashpad集成** | 捕获崩溃，生成minidump |
| **跨平台组件** | 支持DLL/DYLIB、EXE/APP、CEF网页、CEF+嵌入式 |
| **组件化增强** | 标准化接口，易于扩展新组件 |

---

## 二、阶段划分与时间规划

| 阶段 | 任务 | 周期 | 关键产出 |
|------|------|------|----------|
| **阶段0** | 新需求准备 | 1周 | TitleBar接口设计、Crashpad调用 |
| **阶段1** | UI迁移QML | 2-3周 | QML组件、TitleBar双层布局 |

**预计总工期：9-14周**

## 二点五、当前结构盘点与下一步任务

### 当前整体结构

| 模块 | 现状 | 与计划对照 |
|------|------|------------|
| **主程序入口** | `main_qml.cpp` 串联 AppController、ComponentLoader、MainWindowSetup；**未**调用 `CrashpadInit::initialize` | 阶段0 Crashpad 未接入 |
| **AppController** | 有 statusText、pageTitle、hasRunnableComponent、isRunning、theme、registerComponentHost、start/stop 等；**无** componentCount、componentTabs、addComponentTab/switchToTab/closeTab | 阶段0 TitleBar 动态化接口未做 |
| **TitleBar.qml** | 单层 38px，无组件标签栏 | 阶段0/1 双层布局未做 |
| **ComponentLoader** | 已有 ApplicationPaths、ComponentLoader、MainWindowSetup；按 manifest + registerSymbol 加载 DLL，**无** IComponent 标准接口 | 阶段3 组件接口未标准化 |
| **PicMatch 组件** | 有 **PicMatchViewModel**、**PicMatchModel**；**PicMatchWidget** 仍为重量级 QWidget（布局+人脸+播放器）；**PlayerHostItem** 内嵌 QWidget（m_containerWidget + m_picMatchWidget） | 阶段1/2 UI 仍在 Widget，未全 QML |
| **PicMatch QML** | 有 `PicMatchPage.qml`（仅承载 PlayerHostItem）、`FaceShowPanel.qml`、`PicMatchToolBar.qml`、`PicMatchConfigPanel.qml`，但**页面未用** ToolBar/ConfigPanel/FaceShowPanel，仍整块用 Widget | 阶段1 迁移未完成 |
| **Crashpad** | `CrashpadInit.h/cpp` 已实现（USE_CRASHPAD 控制），main 未调用 | 阶段0 差一步 |
| **CEF / 组件接口** | 无 CefManager、CefQuickItem、ComponentInterface | 阶段3/4 未开始 |

### 与阶段目标匹配的「接下来主要任务」

按文档阶段顺序，建议优先做以下任务（可并行或微调顺序）：

| 优先级 | 任务 | 所属阶段 | 关键动作 |
|--------|------|----------|----------|
| **1** | **Crashpad 接入** | 阶段0 | 在 `main_qml.cpp` 最早处 `#include "Common/CrashpadInit.h"` 并 `CrashpadInit::initialize(argc, argv)`；确认 CMake 已选项/打包 crashpad_handler |
| **2** | **TitleBar 动态化接口** | 阶段0 | 在 AppController 增加 componentCount、componentTabs、addComponentTab/removeComponentTab/switchToTab/closeTab；TitleBar.qml 支持 `height: componentCount > 1 ? 78 : 38` 及第二层标签 ListView |
| **3** | **PicMatch 页真正用 QML 组件** | 阶段1 | 重构 `PicMatchPage.qml`：用 ViewModel + PlayerHostItem + PicMatchToolBar + FaceShowPanel + PicMatchConfigPanel，把当前由 PicMatchWidget 承担的工具栏/配置/人脸区逐步迁到 QML，减少 Widget 内 UI |
| **4** | **PlayerHostItem 去 QWidget** | 阶段2 | 播放区改为纯 QQuickItem/OpenGL 渲染，移除 m_containerWidget、PicMatchWidget 的整块嵌入；PicMatchWidget 收敛为仅业务桥接或逐步废弃 |
| **5** | **组件接口标准化** | 阶段3 | 新增 `src/Common/ComponentInterface.h`（如 IComponent：id/name/type、createView、initialize、shutdown），组件加载改为实现该接口 |
| **6** | **CEF 集成** | 阶段4 | 实现 CefManager、CefQuickItem、CefComponent 等（见文档 5.2） |
| **7** | **平台适配与测试** | 阶段5 | 双平台验证、性能与稳定性测试 |

**建议**：先完成 **1 + 2**（阶段0 收尾），再推进 **3 → 4**（先 QML 化再去掉 Widget），这样与「前后端分离 + TitleBar 动态化」目标一致，且每步都可单独验收。

## 三、具体步骤

### 阶段0：新需求准备

#### 0.1 TitleBar动态化
```
步骤：
1. 修改 AppController.h，新增属性和方法
   - componentCount (Q_PROPERTY)
   - componentTabs (Q_PROPERTY)
   - addComponentTab() / removeComponentTab() / switchToTab() / closeTab()

2. 修改 AppController.cpp，实现标签管理逻辑

3. 修改 TitleBar.qml，支持双层布局
   - 第一层：现有标题栏（38px高度）
   - 第二层：组件标签栏（40px高度，仅componentCount>1时显示）
```

#### 0.2 Crashpad集成
```
步骤：
1. 修改 main_qml.cpp 开头

   CrashpadInit::initialize(argc, argv);  // 最先调用

2. 配置CMake，确保crashpad_handler打包到输出目录
```

## 六、QML使用示例

### ViewModel绑定
```qml
import PicMatch 1.0

PicMatchViewModel {
    id: viewModel

    onImageUpdated: (showId, path) => playerHost.updateImage(showId, path)
}

RowLayout {
    PlayerHostItem { ... }

    ColumnLayout {
        PicMatchToolBar {
            running: viewModel.running
            onRunClicked: viewModel.run()
            onStopClicked: viewModel.stop()
        }

        StackLayout {
            FaceShowPanel { faceList: viewModel.faceList }
            PicMatchConfigPanel {
                dataPath: viewModel.dataPath
                onApplyClicked: viewModel.saveConfig()
            }
        }
    }
}
```

### CEF使用
```qml
import CefView 1.0

CefQuickItem {
    url: "http://localhost:8080/app"

    onLoadingFinished: (ok) => console.log("Loaded:", ok)
}
```

## 七、注意事项

1. **渐进式迁移** - 每个阶段保持系统可用
2. **初始化顺序** - Crashpad > CEF > App > 组件
3. **标签唯一性** - 同一组件只能有一个标签
4. **CEF资源路径** - Windows用目录，macOS用.framework
5. **平台差异** - 窗口圆角、进程启动方式不同
