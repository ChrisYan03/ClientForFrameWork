# 📊 当前架构评价

## ✅ 优点

1. **良好的分层设计**
   - 已实现表现层（QML）、桥接层、业务逻辑层的分离
   - 渲染核心使用OpenGL+ImGui，与Qt完全解耦

2. **组件化架构**
   - 插件式动态加载机制（manifest.json + DLL）
   - 组件独立性良好，支持运行时插拔

3. **跨平台支持**
   - 已有macOS/Windows双平台适配
   - CMake配置清晰，平台差异处理得当

4. **现代化UI**
   - 主框架使用纯QML，界面现代
   - 主题系统完善

## ⚠️ 存在的问题

1. **前后端耦合严重**
   ```cpp
   // PicMatchWidget混合了UI和业务逻辑
   - QPushButton/QToolButton等UI控件（应该用QML）
   - OnRunButtonClicked()等业务逻辑
   - applyTheme()等样式管理
   ```

2. **QWidget过度使用**
   - `PicMatchWidget`：整个右侧面板（按钮栏+配置面板）都是QWidget
   - `FaceShowWidget`：人脸显示区域
   - 这些UI控件完全可以改为QML实现

3. **桥接层职责不清**
   - `PlayerHostItem`仅仅是容器，没有提供清晰的前后端接口
   - 业务逻辑直接通过C++回调，没有抽象成Model

4. **配置管理散乱**
   - 配置面板在C++中硬编码
   - 主题样式在C++中管理，应该在QML中声明

5. **TitleBar静态化**
   - 当前TitleBar.qml为静态单层布局
   - 不支持根据组件数量动态调整
   - 缺乏Chrome风格的标签页切换功能

6. **Crashpad未集成**
   - CrashpadInit.h/cpp已实现但未在main_qml.cpp中调用
   - 缺少崩溃捕获机制

### 2.1 重构TitleBar.qml实现动态布局

**设计目标：**
- 根据 `appController.componentCount` 动态调整布局
- 单组件保持现状，多组件显示双层标签栏

**重构后的TitleBar.qml结构：**
```qml
import QtQuick
import QtQuick.Controls
import QtQuick.Layouts
import QtQuick.Window

Rectangle {
    id: titleBarRoot

    // 动态高度：单组件38px，多组件时增加标签栏高度
    height: appController.componentCount > 1 ? 78 : 38

    // 第一层：原有功能（始终存在）
    RowLayout {
        id: firstLayer
        anchors.left: parent.left
        anchors.right: parent.right
        anchors.top: parent.top
        height: 38
        // ... 原有标题栏内容
    }

    // 第二层：组件标签栏（仅多组件时显示）
    Item {
        id: secondLayer
        anchors.left: parent.left
        anchors.right: parent.right
        anchors.top: firstLayer.bottom
        height: 40
        visible: appController.componentCount > 1

        // 标签列表（横向滚动）
        TabBar {
            id: componentTabBar
            anchors.fill: parent
            visible: appController.activeTabs.length > 0

            // 动态生成标签
            Repeater {
                model: appController.activeTabs
                delegate: TabButton {
                    text: modelData.title
                    icon.source: modelData.iconPath
                    checked: appController.currentTabIndex === index
                    onClicked: appController.switchToTab(index)

                    // 关闭按钮
                    Button {
                        id: closeTabBtn
                        icon.source: "qrc:/icons/close.svg"
                        width: 16
                        height: 16
                        onClicked: appController.closeTab(index)
                    }
                }
            }
        }

        // 无标签时的占位提示
        Label {
            anchors.centerIn: parent
            text: qsTr("点击桌面图标打开组件")
            visible: appController.activeTabs.length === 0
            opacity: 0.5
        }
    }
}
```

## 1. TitleBar动态化详细设计

### 1.1 需求规格

| 场景 | 布局 | 说明 |
|------|------|------|
| 组件数 = 1 | 单层 | 保持现有TitleBar样式 |
| 组件数 > 1 | 双层 | 第一层=现有，第二层=标签栏 |

### 1.2 标签栏交互设计

**标签状态：**
- 默认状态：灰色背景 + 图标 + 文字
- 激活状态：蓝色/主题色高亮 + 粗体文字
- Hover状态：浅色背景 + 显示关闭按钮
- 关闭按钮：仅Hover时显示

**交互流程：**
1. 用户在桌面点击组件图标 → 打开组件并创建标签
2. 点击已有标签 → 切换到对应组件
3. 点击关闭按钮 → 关闭组件（但保留标签记录）
4. 再次点击图标 → 重新打开组件（复用或新建标签）

## 2. Crashpad集成详细设计

### 2.1 当前状态分析

**已实现：**
- `src/ClientForFrame/src/Common/CrashpadInit.h` - 接口声明
- `src/ClientForFrame/src/Common/CrashpadInit.cpp` - 完整实现
- 支持Windows/macOS/Linux
- 本地存储minidump

**缺失：**
- 未在main_qml.cpp中调用initialize
- 未配置上报服务器URL
- 未打包crashpad_handler

### 2.2 集成步骤

**Step 1: 修改main_qml.cpp**

```cpp
// src/ClientForFrame/src/main_qml.cpp

#include "Common/CrashpadInit.h"  // 添加头文件

int main(int argc, char *argv[])
{
    // ★★★ 最早调用Crashpad初始化，确保捕获所有崩溃 ★★★
    CrashpadInit::initialize(argc, argv);

#if defined(Q_OS_WIN)
    qputenv("QSG_RHI_BACKEND", "opengl");
#endif
    // ... 后续代码不变
}
```

**Step 2: 配置CMake确保Crashpad依赖**

```cmake
# src/ClientForFrame/CMakeLists.txt 添加
if(USE_CRASHPAD)
    find_package(crashpad REQUIRED)
    target_link_libraries(ClientForFrame PRIVATE crashpad::crashpad_client)
endif()
```

**Step 3: CrashpadHandler打包**

需要在输出目录包含crashpad_handler：
- Windows: `crashpad_handler.exe`
- macOS: `crashpad_handler`
- Linux: `crashpad_handler`

```cmake
# 复制crashpad_handler到输出目录
add_custom_command(TARGET ClientForFrame POST_BUILD
    COMMAND ${CMAKE_COMMAND} -E copy_if_different
        ${CRASHPAD_HANDLER_PATH}
        $<TARGET_FILE_DIR:ClientForFrame>)
```

### 2.3 Crashpad配置选项

**本地存储模式（当前实现）：**
```cpp
// 仅保存minidump到本地
```

**上报服务器模式（可选）：**

### 2.4 崩溃报告查看

**本地存储路径：**
```

│   ├── xxx.dmp
│   └── ...
```

**查看工具：**

```cpp
// AppController.cpp 新增方法实现

void AppController::addComponentTab(const QString& componentId,
                                     const QString& title,
                                     const QString& iconPath)
{
    // 检查是否已存在
    for (const auto& tab : m_componentTabs) {
        if (tab["id"].toString() == componentId) {
            // 已存在，切换到该标签
            int idx = m_componentTabs.indexOf(tab);
            setCurrentTabIndex(idx);
            return;
        }
    }

    // 创建新标签
    QVariantMap newTab;
    newTab["id"] = componentId;
    newTab["title"] = title;
    newTab["iconPath"] = iconPath;
    newTab["isActive"] = true;

    // 取消其他标签的激活状态
    for (auto& tab : m_componentTabs) {
        tab["isActive"] = false;
    }

    m_componentTabs.append(newTab);
    setCurrentTabIndex(m_componentTabs.size() - 1);

    emit componentTabsChanged();
}

void AppController::removeComponentTab(const QString& componentId)
{
    for (int i = 0; i < m_componentTabs.size(); ++i) {
        if (m_componentTabs[i]["id"].toString() == componentId) {
            m_componentTabs.removeAt(i);

            // 如果删除的是当前激活标签，切换到前一个
            if (m_currentTabIndex >= m_componentTabs.size()) {
                setCurrentTabIndex(m_componentTabs.size() - 1);
            }

            emit componentTabsChanged();
            emit tabClosed(componentId);
            return;
        }
    }
}

void AppController::closeTab(const QString& componentId)
{
    // 关闭标签时也要通知组件停止运行
    removeComponentTab(componentId);

    // 通知组件停止
    if (m_componentHost) {
        QMetaObject::invokeMethod(m_componentHost, "quit");
    }
}
```
