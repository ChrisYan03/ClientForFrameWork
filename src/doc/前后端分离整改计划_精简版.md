# 🎯 ClientForFrameWork 前后端分离整改计划

## 一、核心目标

| 目标 | 说明 |
|------|------|
| **前后端分离** | UI全QML，业务逻辑全C++ |
| **TitleBar动态化** | 单组件=单层，多组件=双层（Chrome风格标签） |
| **Crashpad集成** | 捕获崩溃，生成minidump |
| **跨平台组件** | 支持DLL/DYLIB、EXE/APP、CEF网页、CEF+嵌入式 |
| **组件化增强** | 标准化接口，易于扩展新组件 |

## 六、QML使用示例

### ViewModel绑定
```qml
import PicMatch 1.0

PicMatchViewModel {
    id: viewModel

    onImageUpdated: (showId, path) => playerHost.updateImage(showId, path)
}

RowLayout {
    PlayerHostItem { ... }

    ColumnLayout {
        PicMatchToolBar {
            running: viewModel.running
            onRunClicked: viewModel.run()
            onStopClicked: viewModel.stop()
        }

        StackLayout {
            FaceShowPanel { faceList: viewModel.faceList }
            PicMatchConfigPanel {
                dataPath: viewModel.dataPath
                onApplyClicked: viewModel.saveConfig()
            }
        }
    }
}
```

### CEF使用
```qml
import CefView 1.0

CefQuickItem {
    url: "http://localhost:8080/app"

    onLoadingFinished: (ok) => console.log("Loaded:", ok)
}
```

## 七、注意事项

1. **渐进式迁移** - 每个阶段保持系统可用
2. **初始化顺序** - Crashpad > CEF > App > 组件
3. **标签唯一性** - 同一组件只能有一个标签
4. **CEF资源路径** - Windows用目录，macOS用.framework
5. **平台差异** - 窗口圆角、进程启动方式不同
